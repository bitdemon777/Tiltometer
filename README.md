# Poker Tracker

Трекер домашних покерных вечеров: статистика по каждому игроку и подсчёт денег.
Node.js + Express, данные в `data.json`, защита входа общим паролем, отправка
результатов игры на e-mail.

## Возможности

- Вход по одному общему паролю.
- Игроки: добавление, переименование, удаление (если игрок ещё не играл).
- Игра: дата, бай-ин для конкретного вечера, участники с местом, ребаями и аддоном.
- Автоподсчёт денег: взнос игрока = бай-ин + ребаи × бай-ин + аддон; банк вечера — сумма.
- Завершённая игра становится **только для чтения** (историю задним числом не изменить).
- Накопительная статистика и таблица лидеров (по завершённым играм).
- Отправка результатов игры на указанный e-mail.

## Локальный запуск

```bash
npm install
cp .env.example .env   # затем впишите свой APP_PASSWORD (SMTP можно позже)
npm start
```

Откройте http://localhost:3000

## Переменные окружения

| Переменная   | Зачем                                            |
|--------------|--------------------------------------------------|
| `APP_PASSWORD` | Пароль для входа в приложение (обязательно)     |
| `PORT`         | Порт (хостинг обычно задаёт сам)                |
| `SMTP_HOST`    | SMTP-сервер почтового ящика results@…           |
| `SMTP_PORT`    | 465 (SSL) или 587 (STARTTLS)                     |
| `SMTP_USER`    | Логин ящика (results@вашдомен.com)               |
| `SMTP_PASS`    | Пароль ящика                                     |
| `SMTP_FROM`    | Адрес «От кого» (по умолчанию = SMTP_USER)        |

Без SMTP приложение работает полностью — только кнопка «Отправить результат»
сообщит, что отправка не настроена.

## Почта на area.lv

1. В панели area.lv создайте ящик `results@вашдомен.com`.
2. Возьмите его SMTP-параметры (хост вида `mail.вашдомен.com`, порт 465).
3. Впишите их в переменные `SMTP_*` (локально — в `.env`, на хостинге — в настройках окружения).

## Деплой через GitHub

1. Создайте репозиторий на GitHub и запушьте проект (файлы `.env` и `data.json`
   в репозиторий не попадут — они в `.gitignore`).
2. На хостинге с поддержкой Node.js (например, Node.js-приложение на area.lv,
   либо Render/Railway) подключите репозиторий.
3. Команда установки: `npm install`. Команда запуска: `npm start`.
4. Задайте переменные окружения из таблицы выше.
5. Откройте выданный адрес — приложение готово.

> Примечание: на некоторых бесплатных хостингах файловая система сбрасывается при
> перезапуске, и `data.json` может обнулиться. Для постоянного хранения используйте
> хостинг с постоянным диском (area.lv с обычным хостингом подходит).
